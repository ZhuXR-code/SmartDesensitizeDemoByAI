#### 英语 (en)

- **姓名部分遮盖-英语** (ID: 29): 保留英文名字首字母，其余用*代替

#### 德语 (de)

- **姓名部分遮盖-德语** (ID: 32): 保留德文名字首字母，其余用*代替

### 2. 脱敏方式分类

所有规则按照脱敏方式分为三类：

| 脱敏方式 | 说明 | 适用场景 |
|---------|------|---------|
| 🔴 完全遮盖 (full_mask) | 将原始数据完全替换为单个*号 | 密码、密钥等高度敏感信息 |
| 🟢 仿真造数 (simulation) | 生成逼真的假数据，保持格式一致 | 测试数据生成、数据分析 |
| 🟡 部分遮盖 (partial_mask) | 保留部分字符，其余用*代替 | 需要保留部分信息以便识别的场景 |

## 💡 使用建议

### 选择脱敏方式的建议

1. **完全遮盖**
   - 适用于：密码、密钥、CVV码等绝对不能泄露的信息
   - 优点：最安全
   - 缺点：无法保留任何原始信息

2. **仿真造数**
   - 适用于：测试环境、数据分析、模型训练
   - 优点：保持数据格式和真实性
   - 缺点：生成的数据与原始数据无关

3. **部分遮盖**
   - 适用于：用户界面显示、客服场景
   - 优点：便于用户确认身份
   - 缺点：仍可能泄露部分信息

### 多语言数据处理

- **中文数据**：使用 ID 2-21 的规则
- **英文数据**：使用 ID 8, 19, 28-30 的规则
- **日文数据**：使用 ID 9, 24-25 的规则
- **韩文数据**：使用 ID 10, 22-23 的规则
- **法文数据**：使用 ID 11, 26-27 的规则
- **德文数据**：使用 ID 12, 31-32 的规则
