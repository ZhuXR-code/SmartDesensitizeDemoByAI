# 自动识别规则映射修复

## 🐛 问题描述

用户反馈自动识别并脱敏功能存在以下问题：
1. ❌ 银行卡号识别的不对
2. ❌ 姓名识别的不对  
3. ❌ 国家识别的不对
4. ❌ 地址选用的脱敏规则不对

## 🔍 根本原因

**检测规则ID与脱敏规则ID的映射关系完全错误！**

### 错误的映射（修复前）

mapping = {

    2: 4,    # 身份证 → 身份证号仿真 ✓
    16: 15,  # IP地址 → 身份证部分遮盖 ✗ 完全错误！
    3: 19,   # 中文姓名 → 邮箱部分遮盖 ✗ 完全错误！
    4: 5,    # 银行卡号 → 银行卡号仿真 ✓

    5: 6,    # 中文地址 → 地址仿真 ✓

    6: 2,    # 英文姓名 → 姓名仿真(中文) ✗ 应该是英文姓名仿真

## ✅ 修复方案

### 1. 重新建立正确的映射关系

**文件**: `backend/app/services/desensitization_engine.py`

```python
def _find_rule_by_detection(self, match) -> Optional[int]:
    """根据检测结果找到对应的脱敏规则ID"""
    # 检测规则ID → 脱敏规则ID 映射
    mapping = {
        # ========== 手机号 ==========

        19: 3,   # 日本手机号 → 手机号仿真

        25: 3,   # 法国手机号 → 手机号仿真
        28: 3,   # 德国手机号 → 手机号仿真
        
        # ========== 身份证 ==========
        2: 4,    # 中国大陆身份证 → 身份证号仿真
        23: 4,   # 韩国身份证号 → 身份证号仿真
        
        # ========== 姓名 ==========
        3: 2,    # 中文姓名 → 姓名仿真
        6: 8,    # 英文姓名 → 英文姓名仿真
        9: 9,    # 日文姓名 → 日文姓名仿真
        10: 10,  # 韩文姓名 → 韩文姓名仿真
        11: 11,  # 法文姓名 → 法文姓名仿真
        12: 12,  # 德文姓名 → 德文姓名仿真
        
        # ========== 银行卡/信用卡 ==========
        4: 5,    # 中国银行卡号 → 银行卡号仿真
        14: 5,   # 信用卡号 → 银行卡号仿真
        
        # ========== 邮箱 ==========
        7: 19,   # 邮箱地址 → 邮箱部分遮盖
        
        # ========== 地址 ==========
        5: 6,    # 中文地址关键词 → 地址仿真
        18: 6,   # 英文地址关键词 → 地址仿真
        21: 6,   # 日文地址关键词 → 地址仿真
        24: 6,   # 韩文地址关键词 → 地址仿真
        27: 6,   # 法文地址关键词 → 地址仿真
        30: 6,   # 德文地址关键词 → 地址仿真
        
        # ========== 国家 ==========
        # 注意：目前没有专门的国家检测规则
        # 如果有文本被识别为国家，使用默认规则（完全遮盖）
    }
    
    return mapping.get(match.rule_id)
```

### 2. 映射规则设计原则

#### 原则1: 同类数据统一脱敏方式
- **所有手机号** → 手机号仿真（规则ID=3）
- **所有姓名** → 对应语言的姓名仿真
- **所有地址** → 地址仿真（规则ID=6）

#### 原则2: 优先使用仿真造数
- 仿真造数比部分遮盖更安全
- 仿真数据看起来更真实
- 降低数据泄露风险

#### 原则3: 语言匹配
- 中文姓名 → 中文姓名仿真（ID=2）
- 英文姓名 → 英文姓名仿真（ID=8）
- 日文姓名 → 日文姓名仿真（ID=9）
- ...以此类推

#### 原则4: 未匹配的使用默认规则
- 如果检测规则ID不在映射表中
- 返回None，调用方使用默认规则（完全遮盖，ID=1）

## 🔮 未来优化建议

### 1. 添加国家检测规则
```python
{
    "id": 31, 
    "name": "国家名称", 
    "language": "all", 
    "rule_type": "keyword",
    "keywords": ["中国", "美国", "日本", "韩国", "法国", "德国", "英国", "Canada", "Australia"],
    "example": "中国",
    "suggestion": "国家脱敏"
}
```

映射：
```python
31: 7  # 国家名称 → 国家仿真
```

### 2. 添加置信度阈值
```python
if best_match.confidence < 0.6:
    # 低置信度，使用保守的完全遮盖
    return 1
else:
    # 高置信度，使用映射的规则
    return mapping.get(best_match.rule_id)
```

### 3. 支持用户自定义映射
将映射关系存储在数据库中，允许用户调整：
```sql
CREATE TABLE detection_to_desensitization_mapping (
    detection_rule_id INT,
    desensitization_rule_id INT,
    priority INT DEFAULT 0,
    is_active BOOLEAN DEFAULT TRUE
);
```

### 4. 优化地址识别
当前地址识别基于关键词，可能误判。可以改进为：
- 结合列名判断（如列名为"address"、"地址"时提高权重）
- 多关键词组合匹配（如同时包含"省"和"市"）
- 正则表达式匹配完整地址格式
