# 脱敏任务下载功能增强

## 📋 功能概述

为脱敏任务添加了**下载副本**和**下载报告**功能，支持在任务列表和任务详情页面使用。

## ✨ 新增功能

### 1. 下载副本功能

**适用场景：**
- 输出模式为"生成副本"的脱敏任务
- 任务状态为"已完成"

**功能说明：**
- 下载脱敏后的数据文件（CSV格式）
- 文件名格式：`脱敏结果_{任务ID}.csv`
- 支持从任务列表和任务详情页下载

### 2. 下载报告功能

**适用场景：**
- 所有已完成的脱敏任务

**功能说明：**
- 自动生成HTML格式的脱敏报告
- 包含任务基本信息、性能指标、脱敏统计等
- 文件名格式：`脱敏报告_{任务ID}_{时间戳}.html`
- 首次下载时自动生成报告，后续直接下载
- 支持从任务列表和任务详情页下载

## 🎯 技术实现

### 后端实现

#### 1. 数据库模型更新

**文件：** `backend/app/models/desensitization.py`

```python
class DesensitizationTask(Base):
    # ... 其他字段
    report_path = Column(String(500))  # 报告文件路径
```

#### 2. API接口

**生成报告接口：**
```
POST /api/desensitization/tasks/{task_id}/generate-report
```

**下载报告接口：**
```
GET /api/desensitization/tasks/{task_id}/download-report
```

**特点：**
- 下载报告时如果报告不存在，会自动生成
- 报告生成后保存到数据库，避免重复生成
- 报告存储在 `uploads/reports/` 目录

#### 3. 报告内容

生成的HTML报告包含：
- ✅ 报告编号和数据集名称
- ✅ 核心指标概览（准确率、覆盖率、处理速度、总耗时）
- ✅ 性能指标（行数、字段数、识别耗时、脱敏耗时等）
- ✅ 规则覆盖率详情
- ✅ 脱敏准确率详情
- ✅ 优化建议

### 前端实现

#### 1. API封装

**文件：** `frontend/src/api/desensitization.js`

```javascript
// 生成报告
export function generateReport(id) {
  return request.post(`/api/desensitization/tasks/${id}/generate-report`)
}

// 下载报告
export function downloadReport(id) {
  return request.get(`/api/desensitization/tasks/${id}/download-report`, {
    responseType: 'blob'
  })
}
```

## 🎯 功能特点

### 1. 智能报告生成

- **懒加载策略**：只在首次下载时生成报告
- **缓存机制**：报告生成后保存到数据库，避免重复生成
- **自动生成**：下载报告时如果报告不存在，自动触发生成

### 2. 用户体验优化

- **明确提示**：下载成功/失败都有消息提示
- **文件命名**：使用时间戳避免文件名冲突
- **条件显示**：按钮根据任务状态和输出模式智能显示

### 3. 数据完整性

- **双重保障**：检查 output_path 和 temp_file_path
- **状态校验**：只有 completed 状态才能下载
- **异常处理**：完善的错误处理和用户提示

## 📈 报告内容详解

### 核心指标概览

| 指标 | 说明 |
|------|------|
| 脱敏准确率 | 成功脱敏的数据比例 |
| 规则覆盖率 | 配置的规则中被触发的比例 |
| 处理速度 | 每秒处理的行数 |
| 总耗时 | 整个脱敏过程的耗时 |

### 性能指标

- 总数据行数
- 总字段数
- 识别耗时（ms）
- 脱敏耗时（ms）
- 总耗时（ms）
- 处理速度（行/秒）
- 峰值内存（MB）

### 规则覆盖率详情

展示每个规则的：
- 规则名称
- 规则类型
- 匹配次数
- 涉及行数
- 覆盖率

### 脱敏准确率详情

展示每个字段的：
- 字段名称
- 应用规则
- 总数据量
- 已脱敏数量
- 准确率
- 示例（原始值 → 脱敏后值）

### 优化建议

根据分析结果提供改进建议：
- 覆盖率低的规则
- 未触发脱敏的字段
- 脱敏效果优秀的字段

## ⚠️ 注意事项

1. **仅已完成任务可下载**：只有状态为"completed"的任务才能下载副本和报告
2. **副本下载条件**：需要有 output_path 或 temp_file_path
3. **报告自动生成**：首次下载报告时会自动生成，可能需要几秒钟
4. **报告存储位置**：`backend/uploads/reports/` 目录
5. **文件大小**：大型数据集的报告可能较大，下载需要时间
