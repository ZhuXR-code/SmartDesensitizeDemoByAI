## 🔧 技术实现细节

### 数据来源

1. **实时计算的指标**（来自 `ReportGenerator.generate_report()`）
   - 通过 `tracemalloc` 追踪内存使用
   - 通过 `time.perf_counter()` 精确计时
   - 通过 `psutil` 监控系统资源

2. **估算的指标**（来自 API 接口）
   - 识别耗时 = 总耗时 × 30%
   - 脱敏耗时 = 总耗时 × 70%
   - 峰值内存 = 总行数 × 0.001 MB
   - 原因：脱敏任务执行时未记录详细指标

### 计算公式

```python
# 覆盖率计算
overall_coverage = (已覆盖的规则数 / 配置的规则总数) × 100%

# 准确率计算
overall_accuracy = Σ(各字段准确率) / 字段总数

# 处理速度
rows_per_second = 总行数 / (总耗时 / 1000)

# 敏感字段统计
sensitive_fields = set(所有匹配结果的column_name)
sensitive_fields_count = len(sensitive_fields)

# 总匹配次数
total_matches = Σ(各规则的matched_count)
```
