# 银行卡识别优化 - Luhn算法与BIN码匹配

## 📋 问题描述

原有的银行卡识别规则（ID=4）仅使用简单的正则表达式`\d{16,19}`匹配16-19位数字，导致：
- ❌ 误识别：身份证号、订单号等长数字被当作银行卡
- ❌ 准确率低：无法区分真正的银行卡号和普通数字串
- ❌ 无法识别银行：不知道是哪个银行的卡

## ✅ 解决方案

引入**Luhn算法校验**和**BIN码匹配**，大幅提升银行卡识别的准确性。

### 核心改进

1. **Luhn算法校验**：验证卡号的数学合法性
2. **BIN码识别**：根据前6位识别银行和卡类型
3. **动态置信度**：Luhn校验通过则提高置信度到0.95
4. **智能过滤**：Luhn校验失败则直接跳过，避免误识别

## 🔧 技术实现

### 1. Luhn算法实现

**文件**: `backend/app/services/detection_engine.py`

```python
@staticmethod
def luhn_check(card_number: str) -> bool:
    """
    Luhn算法校验银行卡号
    
    算法步骤：
    1. 从右往左，偶数位数字乘以2
    2. 如果乘以2的结果大于9，则减去9
    3. 所有数字求和
    4. 如果总和能被10整除，则校验通过
    """
    if not card_number or not card_number.isdigit():
        return False
    
    digits = [int(d) for d in card_number]
    total = 0
    
    for i, digit in enumerate(reversed(digits)):
        if i % 2 == 1:  # 偶数位（从0开始计数）
            doubled = digit * 2
            if doubled > 9:
                doubled -= 9
            total += doubled
        else:
            total += digit
    
    return total % 10 == 0
```

### 3. 综合判断

```python
def is_bank_card(self, card_number: str) -> Tuple[bool, str, Optional[Dict]]:
    """综合判断是否为银行卡"""
    
    # 1. 去除空格和横线
    clean_number = re.sub(r'[\s\-]', '', card_number)
    
    # 2. 长度检查（16-19位）
    if len(clean_number) < 16 or len(clean_number) > 19:
        return False, "长度不符合", None
    
    # 3. 纯数字检查
    if not clean_number.isdigit():
        return False, "包含非数字字符", None
    
    # 4. BIN码识别
    bank_info = self.identify_bank_bin(clean_number)
    
    # 5. Luhn校验
    if not self.luhn_check(clean_number):
        return False, "Luhn校验失败", bank_info
    
    # 6. 综合判断
    if bank_info:
        return True, f"识别为{bank_info['bank']}{bank_info['type']}", bank_info
    else:
        return True, "Luhn校验通过，疑似银行卡", None
```
