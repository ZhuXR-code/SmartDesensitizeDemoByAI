## 一、项目概述（30秒）

尊敬的评委老师，大家好！

我们团队带来的项目是 **"敏感信息智能脱敏平台"** —— 一款面向数据安全领域的智能化解决方案。

在数字化转型浪潮中，数据已成为核心资产，但敏感信息泄露事件频发。我们的平台致力于解决一个核心痛点：**如何在保护隐私的前提下，让数据继续发挥价值？**

---

## 二、问题背景（1分钟）

### 2.1 现实困境

| 场景 | 痛点 |
|------|------|
| 开发测试 | 开发人员需要真实数据，但生产数据含大量敏感信息 |
| 数据共享 | 企业与第三方合作时，数据脱敏处理耗时耗力 |
| 合规要求 | GDPR、个人信息保护法等法规对数据安全提出严格要求 |
| 多语言环境 | 跨国企业需要处理中、英、日、韩等多语言敏感数据 |

### 2.2 现有方案不足

- **手动处理**：效率低、易遗漏、不可复现
- **简单替换**：破坏数据格式和关联性
- **单一规则**：无法适应多语言、多场景的复杂需求
- **黑盒操作**：用户无法预览和确认脱敏效果

## 三、核心亮点（2分钟）

### 亮点一：多语言智能识别引擎

**技术创新点**：
- 支持 **6 种语言** 的敏感数据识别：中文、英文、日文、韩文、法文、德文
- 基于字符集特征的语言自动检测算法
- 每种语言配备专属识别规则库

**实际价值**：
- 跨国企业无需为每种语言单独配置
- 混合语言数据也能精准识别

> 演示数据：一份包含中、英、日、韩、法、德六种语言姓名的表格，系统一次性全部识别。

### 亮点二：识别-脱敏一体化流水线

**技术创新点**：
- 识别结果 **一键跳转** 脱敏模块
- 自动传递数据快照和敏感列信息
- 智能推荐每列最优脱敏规则

**实际价值**：
- 用户无需重复配置
- 从识别到脱敏，三步完成
- 大幅降低操作门槛

### 亮点三：关联仿真脱敏技术

**技术创新点**：
- 采用 **确定性仿真算法**，相同原始数据 + 相同密钥 = 相同脱敏结果
- 支持 30 组独立密钥，用户仅见别名
- 保持跨表数据关联性

**实际价值**：
- 脱敏后的数据仍可用于关联分析
- 满足数据分析场景的需求
- 密钥安全隔离，防止逆向破解

### 亮点四：可视化前后对比确认

**技术创新点**：
- 脱敏前展示 **10-20 条数据的前后对比**
- 用户确认后才执行全量处理
- 高危操作（覆盖原数据）红色警告 + 二次确认

**实际价值**：
- 避免误操作导致数据损坏
- 非技术人员也能放心使用
- 符合企业安全审计要求

### 亮点五：四种页面风格自由切换

**技术创新点**：
- 支持 **经典、Vue经典、暗紫、黑金** 四种主题风格
- 一键切换，实时生效，无需刷新页面
- 基于 CSS 变量和 SCSS 的主题系统，扩展性强
- 用户偏好自动保存到本地存储

**实际价值**：
- 经典风格：蓝白商务风，清晰明亮，适合日常办公
- Vue经典风格：Element Plus 原生风格，适合开发调试
- 暗紫风格：暮色紫雾玻璃质感，降低屏幕亮度刺激，适合夜间使用
- 黑金风格：青苔紫夜奢华，沉稳大气，适合演示汇报
- 满足不同用户的审美偏好和使用场景

### 亮点六：AI 大模型智能识别与脱敏

**技术创新点**：
- 集成 **DeepSeek / OpenAI / 千问 / Kimi / GLM / 文心 / Azure** 等多种大语言模型
- 基于 LLM 的**语义级敏感判断**，准确率 95%+，识别规则引擎难以捕捉的隐式敏感数据
- **人工复核机制**：AI 识别结果逐条可确认/否定，复核结果驱动脱敏
- AI 推理过程可追溯，附带法规引用（PBoC、NFRA 等）

**实际价值**：
- 规则引擎 + AI 引擎双保险，全面覆盖敏感信息
- 准确率远超纯规则方案，减少误报漏报
- 人工复核提供最终决策权，满足合规审计要求
- 支持自定义/私有化部署模型，数据不出域

### 亮点七：快速脱敏工作流

**技术创新点**：
- **引导式4步线性流程**：导入数据 → 识别敏感 → 配置脱敏 → 报告导出
- 自动进度跟踪与步骤解锁机制
- 进度数据本地持久化，关闭页面不丢失

**实际价值**：
- 新用户无需学习，按步骤操作即可完成脱敏
- 大幅降低使用门槛，非技术人员也能独立完成
- 操作步骤清晰可追溯

### 亮点八：前后端分离 + 多种部署方案

**技术创新点**：
- 前端 Vue3 + 后端 FastAPI 完全分离
- 提供完整的 RESTful API 文档（Swagger）
- CORS 跨域支持，易于第三方集成
- 三种部署方案：Docker 一键部署 / 便携式部署(SQLite) / Windows 安装包

**实际价值**：
- 可适配不同前端框架
- 可集成到企业现有系统
- Docker 方案：一条命令完成部署
- 便携方案：无需 MySQL，U盘即插即用
- 安装包方案：小白用户双击安装即可

                            │
┌─────────────────────────────────────────────────────────┐
│                      API 网关 (FastAPI)                   │

└─────────────────────────────────────────────────────────┘
                            │
┌─────────────────────────────────────────────────────────┐

### 技术选型理由

| 层级 | 技术 | 选择理由 |
|------|------|----------|
| 前端 | Vue 3 + Element Plus | 组件丰富、响应式、生态成熟、四套主题 |
| 后端 | FastAPI | 异步高性能、自动 API 文档、类型安全 |
| AI 引擎 | 多模型统一 API 调用 | 支持主流大模型，按需切换 |
| 数据库 | SQLAlchemy ORM | 支持 SQLite/MySQL 无缝切换 |
| 数据处理 | Pandas | 强大的数据处理能力，支持多种格式 |

## 五、功能演示（2分钟）

### 场景一：某电商公司需要给外包团队提供用户数据用于测试

**Step 1 - 导入数据**
> 演示：上传包含用户信息的 CSV 文件，或连接 MySQL 数据库选择用户表

**Step 2 - 识别敏感信息**
> 演示：创建识别任务，系统自动发现：
> - 中文姓名、手机号、身份证号
> - 英文姓名、邮箱、信用卡号
> - 地址信息
> 展示识别报告：类型分布饼图、每列密度柱状图

**Step 3 - 脱敏处理**
> 演示：一键跳转脱敏，系统自动推荐规则：
> - 姓名 → 仿真替换
> - 手机号 → 部分遮盖
> - 身份证号 → 部分遮盖
> - 地址 → 部分遮盖
> 展示前后对比预览，确认后执行

**Step 4 - 输出结果**
> 演示：下载脱敏后的文件，数据格式完整保留，敏感信息已安全处理

### 场景二：AI 大模型智能识别复杂语义敏感信息

**Step 1 - 配置 AI 模型**
> 演示：进入 AI 配置，选择 DeepSeek 模型，填写 API Key，测试连接成功

**Step 2 - AI 智能识别**
> 演示：选择同一份数据，启动 AI 识别。LLM 逐行分析，发现规则引擎遗漏的隐式敏感数据：
> - "王总上个月转账了50万" → 识别为敏感金额信息
> - "患者初步诊断为糖尿病" → 识别为医疗健康信息
> 展示 AI 推理过程和置信度评分

**Step 3 - 人工复核确认**
> 演示：逐条复核 AI 结果，确认/否定，填写复核理由。支持一键全部采纳

**Step 4 - AI 脱敏与报告**
> 演示：基于复核结果执行 AI 脱敏，选择仿真模式，生成脱敏报告

## 六、创新性与优势（1分钟）

### 6.1 与竞品对比

| 特性 | 传统工具 | 我们的平台 |
|------|----------|-----------|
| 多语言支持 | 单一语言 | **6 种语言自动识别** |
| AI 智能识别 | 无 | **大模型语义识别 + 人工复核** |
| 识别-脱敏流程 | 分离操作 | **一键流水线 + 引导式工作流** |
| 脱敏算法 | 简单替换 | **关联仿真，保持数据关联** |
| 效果确认 | 无预览 | **可视化前后对比** |
| 扩展性 | 封闭系统 | **开放 API，易于集成** |
| 页面风格 | 单一固定 | **四种风格自由切换** |
| 数据源 | 仅文件 | **文件 + 数据库直连** |
| 部署方式 | 复杂安装 | **Docker/便携/安装包三种方案** |

### 6.2 核心创新点总结

1. **AI 大模型驱动**：语义级敏感判断，识别传统规则无法发现的隐式敏感信息
2. **双引擎架构**：规则引擎 + AI 引擎双重保障，覆盖不同精度需求
3. **人工复核闭环**：AI 识别 → 人工确认 → 驱动脱敏，合规可审计
4. **多语言混合识别**：突破单一语言限制，适应全球化场景
5. **智能规则推荐**：基于列名和识别结果自动推荐脱敏策略
6. **关联保持脱敏**：确定性算法确保跨表数据一致性
7. **安全确认机制**：预览确认 + 高危警告，防止误操作
8. **四种风格切换**：经典/Vue经典/暗紫/黑金主题，满足不同场景需求
9. **开放架构设计**：前后端分离，API 文档完善，易于生态扩展
