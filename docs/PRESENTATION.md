## 一、项目概述（30秒）

尊敬的评委老师，大家好！

我们团队带来的项目是 **"敏感信息智能脱敏平台"** —— 一款面向数据安全领域的智能化解决方案。

在数字化转型浪潮中，数据已成为核心资产，但敏感信息泄露事件频发。我们的平台致力于解决一个核心痛点：**如何在保护隐私的前提下，让数据继续发挥价值？**

---

## 二、问题背景（1分钟）

### 2.1 现实困境

| 场景 | 痛点 |
|------|------|
| 开发测试 | 开发人员需要真实数据，但生产数据含大量敏感信息 |
| 数据共享 | 企业与第三方合作时，数据脱敏处理耗时耗力 |
| 合规要求 | GDPR、个人信息保护法等法规对数据安全提出严格要求 |
| 多语言环境 | 跨国企业需要处理中、英、日、韩等多语言敏感数据 |

### 2.2 现有方案不足

- **手动处理**：效率低、易遗漏、不可复现
- **简单替换**：破坏数据格式和关联性
- **单一规则**：无法适应多语言、多场景的复杂需求
- **黑盒操作**：用户无法预览和确认脱敏效果

---

## 三、核心亮点（2分钟）

### 亮点一：多语言智能识别引擎

**技术创新点**：
- 支持 **6 种语言** 的敏感数据识别：中文、英文、日文、韩文、法文、德文
- 基于字符集特征的语言自动检测算法
- 每种语言配备专属识别规则库

**实际价值**：
- 跨国企业无需为每种语言单独配置
- 混合语言数据也能精准识别

> 演示数据：一份包含中、英、日、韩、法、德六种语言姓名的表格，系统一次性全部识别。

### 亮点二：识别-脱敏一体化流水线

**技术创新点**：
- 识别结果 **一键跳转** 脱敏模块
- 自动传递数据快照和敏感列信息
- 智能推荐每列最优脱敏规则

**实际价值**：
- 用户无需重复配置
- 从识别到脱敏，三步完成
- 大幅降低操作门槛

### 亮点三：关联仿真脱敏技术

**技术创新点**：
- 采用 **确定性仿真算法**，相同原始数据 + 相同密钥 = 相同脱敏结果
- 支持 30 组独立密钥，用户仅见别名
- 保持跨表数据关联性

**实际价值**：
- 脱敏后的数据仍可用于关联分析
- 满足数据分析场景的需求
- 密钥安全隔离，防止逆向破解

### 亮点四：可视化前后对比确认

**技术创新点**：
- 脱敏前展示 **10-20 条数据的前后对比**
- 用户确认后才执行全量处理
- 高危操作（覆盖原数据）红色警告 + 二次确认

**实际价值**：
- 避免误操作导致数据损坏
- 非技术人员也能放心使用
- 符合企业安全审计要求

### 亮点五：三种页面风格自由切换

**技术创新点**：
- 支持 **经典、暗紫、黑金** 三种主题风格
- 一键切换，实时生效，无需刷新页面
- 基于 CSS 变量和 SCSS 的主题系统，扩展性强
- 用户偏好自动保存到本地存储

**实际价值**：
- 经典风格：清晰明亮，适合日常办公
- 暗紫风格：降低屏幕亮度刺激，适合夜间使用
- 黑金风格：沉稳大气，适合演示汇报
- 满足不同用户的审美偏好和使用场景

### 亮点六：前后端分离 + API 开放架构

**技术创新点**：
- 前端 Vue3 + 后端 FastAPI 完全分离
- 提供完整的 RESTful API 文档
- CORS 跨域支持，易于第三方集成

**实际价值**：
- 可适配不同前端框架
- 可集成到企业现有系统
- 可对接 AI 大模型进行智能化升级

## 四、技术架构（1分钟）

```
┌─────────────────────────────────────────────────────────┐
│                      前端层 (Vue3)                        │
│  仪表盘 │ 数据集管理 │ 识别模块 │ 脱敏模块 │ 管理模块      │
└─────────────────────────────────────────────────────────┘
                            │
┌─────────────────────────────────────────────────────────┐
│                      API 网关 (FastAPI)                   │
│  统一响应格式 │ 自动文档 │ 数据验证 │ 异常处理            │
└─────────────────────────────────────────────────────────┘
                            │
┌─────────────────────────────────────────────────────────┐
│                      服务层                               │
│  语言检测引擎 │ 敏感识别引擎 │ 脱敏处理引擎 │ 数据服务      │
└─────────────────────────────────────────────────────────┘
                            │
┌─────────────────────────────────────────────────────────┐
│                      数据层                               │
│  SQLite (开发) / MySQL (生产) │ 文件存储 │ 数据库连接      │
└─────────────────────────────────────────────────────────┘
```

### 技术选型理由

| 层级 | 技术 | 选择理由 |
|------|------|----------|
| 前端 | Vue 3 + Element Plus | 组件丰富、响应式、生态成熟 |
| 后端 | FastAPI | 异步高性能、自动 API 文档、类型安全 |
| 数据库 | SQLAlchemy ORM | 支持 SQLite/MySQL 无缝切换 |
| 数据处理 | Pandas | 强大的数据处理能力，支持多种格式 |

## 五、功能演示（2分钟）

### 场景：某电商公司需要给外包团队提供用户数据用于测试

**Step 1 - 导入数据**
> 演示：上传包含用户信息的 CSV 文件，或连接 MySQL 数据库选择用户表

**Step 2 - 识别敏感信息**
> 演示：创建识别任务，系统自动发现：
> - 中文姓名、手机号、身份证号
> - 英文姓名、邮箱、信用卡号
> - 地址信息
> 展示识别报告：类型分布饼图、每列密度柱状图

**Step 3 - 脱敏处理**
> 演示：一键跳转脱敏，系统自动推荐规则：
> - 姓名 → 仿真替换
> - 手机号 → 部分遮盖
> - 身份证号 → 部分遮盖
> - 地址 → 部分遮盖
> 展示前后对比预览，确认后执行

**Step 4 - 输出结果**
> 演示：下载脱敏后的文件，数据格式完整保留，敏感信息已安全处理

## 六、创新性与优势（1分钟）

### 6.1 与竞品对比

| 特性 | 传统工具 | 我们的平台 |
|------|----------|-----------|
| 多语言支持 | 单一语言 | **6 种语言自动识别** |
| 识别-脱敏流程 | 分离操作 | **一键流水线** |
| 脱敏算法 | 简单替换 | **关联仿真，保持数据关联** |
| 效果确认 | 无预览 | **可视化前后对比** |
| 扩展性 | 封闭系统 | **开放 API，易于集成** |
| 页面风格 | 单一固定 | **三种风格自由切换** |
| 数据源 | 仅文件 | **文件 + 数据库直连** |

### 6.2 核心创新点总结

1. **多语言混合识别**：突破单一语言限制，适应全球化场景
2. **智能规则推荐**：基于列名和识别结果自动推荐脱敏策略
3. **关联保持脱敏**：确定性算法确保跨表数据一致性
4. **安全确认机制**：预览确认 + 高危警告，防止误操作
5. **三种风格切换**：经典/暗紫/黑金主题，满足不同场景需求
6. **开放架构设计**：前后端分离，API 文档完善，易于生态扩展
