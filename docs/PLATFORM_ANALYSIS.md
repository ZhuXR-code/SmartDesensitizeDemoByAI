### 3.2 确定性脱敏引擎（关联仿真核心原理）

```
原始数据A ──→ AES-256-ECB 加密 ──→ 密文
                 │
                 └── [密钥K]
                          │
                          ▼
                    密文 ──→ SHA-256 ──→ 64位锚点值
                                            │
                                            ▼
                                    SplitMix64 PRNG
                                    (确定性随机序列)
                                            │
                          ┌─────────────────┼─────────────────┐
                          ▼                 ▼                  ▼
                    姓名仿真表     手机号仿真表      身份证仿真表
                    (基于锚点映射)  (锚点→号码)     (锚点→证件号)
                          │                 │                  │
                          └─────────────────┼──────────────────┘
                                            │
                                            ▼
                                    脱敏后的伪造数据

数学保证：
  Mask(原文, 密钥, 算法) → 始终相同的输出
  相同原文 + 相同密钥 → 必然相同脱敏结果
  → 跨表关联一致性

不同密钥组：
  Key_A → 脱敏结果集A (业务部门A)
  Key_B → 脱敏结果集B (业务部门B)
  → 不同部门间数据完全隔离，无法碰撞还原
```

```
数据集 ──→ 逐行逐列遍历
                │
                ▼
        构造AI提示词
        (字段名 + 值 + 法规背景 + 可选联网搜索)
                │
                ▼

        (OpenAI / DeepSeek / Qwen / Kimi / GLM...)
                │
                ▼

        { is_sensitive, sensitive_type, confidence, risk_level, regulation_ref, reasoning }
                │
                ▼
        存储 AI识别结果
                │
                ▼
    ┌─────────────────────────────────┐
    │        人工复核阶段              │
    │                                 │
    │  逐条展示 AI判断 + AI推理过程    │
    │       ↙              ↘         │
    │  确认(敏感)        否定(非敏感)  │
    │       ↓              ↓         │
    │  记录复核理由     记录复核理由   │
    │       ↓              ↓         │

             │              │
             ▼              ▼
```

| 数据接入 | ★★★★★ | 文件上传 + 数据库连接 + 剪贴板，覆盖主流数据来源 |
| 识别能力 | ★★★★★ | 正则引擎 + AI语义引擎双轨，支持6语言，地址精细解析 |
| 脱敏策略 | ★★★★★ | 5种策略，30组密钥，确定性加密保证跨表一致 |
| 报告体系 | ★★★★☆ | HTML/MD/JSON多格式，含领导视图 |
| AI集成 | ★★★★★ | 多模型支持，识别+脱敏+关联仿真全覆盖 |
| 主题体验 | ★★★★☆ | 四套主题，响应式布局 |
| CLI工具 | ★★★☆☆ | 命令行脱敏工具，但功能较基础 |
| 部署运维 | ★★★★☆ | Docker化部署，SVG大屏 |

### 4.2 核心优势

1. **双引擎识别**：正则引擎（速度快、成本低） + AI引擎（语义准确、法规引用），形成互补
2. **关联仿真脱敏**：独创的确定性脱敏算法（AES-256 + SHA-256 + SplitMix64），保证跨表关联一致性
3. **零代码操作**：全图形化界面，从导入到脱敏只需要三步
4. **人工复核机制**：AI识别结果允许人工确认/否定，以人审结果驱动脱敏
5. **多模型AI接入**：支持8+种AI模型，包括国产DeepSeek/Qwen/GLM/文心

### 4.3 可改进点

1. **AI任务进度不可靠**：轮询方式获取进度，大量并发时数据库压力大，建议改用WebSocket推送
2. **数据预览限制**：大文件（>100MB）预览可能内存溢出，建议实现分片预览
3. **脱敏规则可视化不足**：规则之间的执行顺序和冲突无法直观看到
4. **缺少审计日志**：缺少独立的操作审计追踪记录
5. **缺少团队协作**：单用户模式，缺少多用户/角色/权限管理
6. **缺少API开放文档**：仅Swagger文档，缺少第三方集成指导

## 五、工作流/操作流程优化建议

### 5.1 当前操作流程问题

当前平台各功能模块虽然完整，但页面之间是**松散的菜单导航**，用户需要：

- 记住每个子菜单的路径
- 手动在"数据集 → 识别 → 脱敏 → 报告"之间跳转
- 对于新用户，理解操作顺序有门槛

## 六、总结

这是一个功能相当完整的数据安全平台，核心价值链路清晰：**数据导入 → 识别 → 脱敏 → 报告**。平台在以下几个方面表现突出：

1. **双引擎架构**（正则+AI）兼顾了性能与准确性
2. **独创的确定性脱敏算法**解决了跨表关联一致性的行业痛点
3. **多模型AI集成**在国内数据安全工具中较为领先

建议优先实现**引导式6步工作流**，这将显著降低新用户的使用门槛，让"识别→脱敏→报告"这一核心价值链路对用户更加直观可见。
