## 0. 快速脱敏工作流（推荐）

> 这是平台新增的**引导式操作流程**，将导入数据 → 敏感识别 → 脱敏执行 → 报告导出四个步骤串联为一条线性流程，推荐所有用户优先使用。

### 访问方式

- **侧边栏菜单**：点击左侧菜单 **⚡ 快速脱敏**，仅次于"首页"菜单
- **首页入口**：登录后在首页可以看到"快速脱敏工作流"卡片，点击 **进入快速脱敏**

### 工作流4步骤

```
步骤1                   步骤2                   步骤3                   步骤4
┌────────────┐       ┌────────────┐          ┌────────────┐          ┌────────────┐
│  导入数据   │  →   │  识别敏感   │    →     │  配置脱敏   │   →     │ 报告与导出  │
│            │       │   数据     │          │   并执行    │          │            │
│ 文件/数据库 │       │ 规则/AI引擎 │         │ 5种策略可选 │         │ HTML/MD   │
└────────────┘       └────────────┘          └────────────┘          └────────────┘
```

### 各步骤详情

**步骤1：导入数据**
- 支持 Excel (.xlsx/.xls)、CSV、TXT、JSON、Markdown、日志文件上传
- 支持 MySQL / PostgreSQL / Oracle / SQL Server 数据库连接导入
- 支持剪贴板粘贴导入
- 文件最大 100MB，自动识别编码和格式

**步骤2：识别敏感数据**
- **规则引擎识别**：基于正则+关键词，支持6种语言（中/英/日/韩/法/德）
- **AI智能识别**：基于大模型的语义级敏感判断，准确率95%+
- 自动识别姓名、身份证号、手机号、银行卡号、地址、邮箱等20+类敏感数据

**步骤3：配置脱敏并执行**
- 5种脱敏策略：完全遮盖、部分遮盖、仿真造数、关联仿真、固定遮盖
- 智能推荐：根据识别结果自动匹配最优脱敏规则
- 预览确认：执行前展示10-20条脱敏前后对比数据
- 关联仿真：基于密钥的确定性脱敏，保证跨表数据一致性

**步骤4：查看报告与导出**
- HTML 在线预览：在浏览器中直接查看脱敏前后对比
- HTML 下载：完整的脱敏报告，含统计图表和数据表格
- Markdown 下载：适合版本管理和开发人员使用
- 脱敏数据文件下载：支持 XLSX 和 CSV 两种格式

### 进度跟踪

- 工作流页面会自动跟踪您的完成进度
- 每完成一个步骤并返回工作流页面，该步骤自动标记为 ✅ 已完成
- 已完成和当前可执行的步骤可以点击跳转，未解锁的步骤不可点击
- 进度数据保存在浏览器本地，关闭页面后重新打开进度依然保留

## 1. 快速开始：三步完成脱敏

### 第一步：导入数据

1. 打开浏览器，访问平台地址（如 `http://localhost:8080`）
2. 点击左侧菜单 **📄 数据集管理** → **导入数据**
3. 在页面中选择导入方式：
   - **文件上传**：点击虚线框选择文件，或直接拖拽 Excel/CSV/TXT/JSON 文件到上传区
   - **数据库连接**：输入数据库地址、用户名、密码，点击"测试连接"，然后选择表导入
   - **剪贴板粘贴**：从 Excel 复制数据后粘贴
4. 文件上传后系统会自动识别格式和编码
5. 输入数据集名称（如"客户信息表"），点击 **确认导入**

> ✅ 成功标志：页面提示"导入成功"，数据集列表中出现新记录

### 第二步：识别敏感信息

1. 点击左侧菜单 **🔍 敏感数据识别** → **识别任务**
2. 点击右上角 **➕ 创建任务** 按钮
3. 填写任务信息：
   - **选择数据集**：下拉选择刚导入的数据
   - **选择规则集**：选择"001"或"002"规则集
   - **语言策略**：推荐选"自动检测"
4. 点击 **开始识别**
5. 等待进度条到 100%，状态变为"completed"
6. 点击 **详情** 查看识别结果：
   - 哪些列包含敏感数据
   - 每条敏感数据的类型和位置
   - 系统推荐的脱敏方式

> ✅ 成功标志：状态变为"completed"，可以看到敏感信息分布饼图

### 第三步：执行脱敏

1. 在识别结果页或任务列表中，点击 **→ 去脱敏** 按钮
2. 系统自动跳转到脱敏任务创建页，自动加载列和推荐规则
3. 检查每列的脱敏规则是否合适（可以手动更换）
4. 点击 **智能推荐** 可一键应用最优规则
5. 点击 **下一步：预览确认**
6. 查看前后对比（10-20条示例数据）
7. 确认无误后，选择输出方式：
   - **生成副本**（推荐）：保留原数据，生成脱敏后的新文件
   - **覆盖原数据**（⚠️ 需二次确认）：直接修改原始数据
8. 点击 **开始脱敏**，等待完成
9. 在脱敏任务列表中可以 **下载副本** 或 **预览/下载报告**

> 🎉 恭喜！您已完成第一次脱敏操作！

## 3. 数据集管理

### 3.1 文件上传

**支持的格式和大小：**

| 格式 | 扩展名 | 最大大小 |
|------|--------|---------|
| Excel | .xlsx, .xls | 100MB |
| CSV | .csv | 100MB |
| 文本 | .txt | 100MB |
| JSON | .json | 100MB |
| Markdown | .md | 100MB |
| 日志 | .log | 100MB |

**操作步骤：**

1. 进入 **数据集管理** → **导入数据**
2. 默认显示"文件上传"标签页
3. 点击虚线框区域，在弹出文件对话框中选择文件，或直接从文件夹拖拽文件到此处
4. 系统自动识别：
   - 文件格式（Excel/CSV/JSON等）
   - 字符编码（UTF-8/GBK等）
   - 列名和数据类型
5. 输入数据集名称（或使用自动生成的文件名）
6. 点击 **确认导入**

> 💡 **提示：** 如果文件是乱码，请手动选择正确的编码（如 GBK）

### 3.2 数据库连接

**支持的数据库：**

| 数据库 | 默认端口 |
|--------|---------|
| MySQL | 3306 |
| PostgreSQL | 5432 |
| Oracle | 1521 |
| SQL Server | 1433 |

**操作步骤：**

1. 进入 **数据集管理** → **导入数据** → **数据库连接**
2. 输入连接信息：
   - 数据源名称（自定义，如"生产库-客户"）
   - 数据库类型（下拉选择）
   - 主机地址（如 `192.168.1.100`）
   - 端口（如 `3306`）
   - 数据库名
   - 用户名和密码
3. 点击 **🔗 测试连接**
4. 连接成功后，点击 **📋 加载表列表**
5. 勾选需要导入的表（支持多选）
6. 可设置数据集名称前缀（如"生产_"）
7. 点击 **导入选中表到数据集**

> ⚠️ **安全提醒：** 建议为平台创建数据库只读账号，限定访问范围

### 3.3 管理数据集

在 **数据集列表** 中可以：
- 🔍 查看数据集详情（行数、列数、大小、编码）
- 🗑️ 删除不需要的数据集（确认后不可恢复）
- 📋 复制数据集名称

## 4. 敏感数据识别

### 4.1 支持的数据类型

**中文：**

| 敏感类型 | 示例 | 识别规则 |
|---------|------|---------|
| 姓名 | 张三、李小明 | 中文姓名正则 + 关键词 |
| 手机号 | 13800138000 | 11位手机号正则 |
| 身份证号 | 320102199001011234 | 18位身份证格式+校验码 |
| 银行卡号 | 6222021234567890123 | 16-19位银行卡格式 |
| 地址 | 北京市朝阳区xxx | 地址关键词 + 层级匹配 |
| 邮箱 | abc@example.com | 邮箱正则 |

**多语言支持（自动检测）：**

| 语言 | 可识别的敏感类型 |
|------|----------------|
| 🇨🇳 中文 | 姓名、手机号、身份证号、银行卡号、地址、邮箱 |
| 🇬🇧 英文 | 姓名、手机号、邮箱、信用卡号、护照号 |
| 🇯🇵 日文 | 姓名、手机号、地址 |
| 🇰🇷 韩文 | 姓名、手机号 |
| 🇫🇷 法文 | 姓名、邮箱 |
| 🇩🇪 德文 | 姓名、邮箱 |

### 4.2 创建识别任务

1. 进入 **敏感数据识别** → **识别任务**
2. 点击 **➕ 创建任务**
3. 填写配置：
   - **任务名称**：可选填，如"客户数据月度审查"
   - **选择数据集**：从下拉列表选择已导入的数据集
   - **选择规则集**：选择识别规则组合
   - **扫描列**：选择要扫描的列（默认全部）
   - **语言策略**：
     - `自动检测`（推荐）— 系统自动判断每行数据语言
     - `指定语言` — 强制使用某种语言规则
4. 点击 **开始识别**

### 4.3 查看识别报告

识别完成后，点击任务 **详情** 查看：

1. **敏感类型分布饼图** — 一目了然哪些类型的敏感数据最多
2. **每列敏感密度柱状图** — 哪一列表格最敏感
3. **明细列表** — 每条敏感数据的位置
4. **脱敏建议** — 系统推荐每列应该如何脱敏

### 4.4 直接跳转脱敏

在识别结果页或任务列表，点击 **→ 去脱敏** 按钮：

- 系统自动将识别出的敏感列和推荐规则传递给脱敏任务
- 无需重复配置
- 直接进入脱敏预览环节

## 5. 数据脱敏

### 5.1 脱敏规则类型

| 规则类型 | 说明 | 示例 |
|---------|------|------|
| 🎭 仿真 | 生成格式相同、数值合理的假数据 | 张三 → 李四 |
| 🔗 关联仿真 | 相同原始值 → 相同假数据（跨表一致） | 表A张三→李四 = 表B张三→李四 |
| 🖊️ 部分遮盖 | 保留首尾、中间打星号 | 138****8000 |
| ██ 完全遮盖 | 全部替换为星号 | 张三 → ** |

### 5.2 创建脱敏任务（两种方式）

**方式一：从识别结果跳转 ⭐（推荐）**

1. 在识别任务列表，找到已完成的任务
2. 点击 **→ 去脱敏**
3. 系统自动加载数据集和推荐规则
4. 核对或调整规则后，直接进入预览

**方式二：手动创建**

1. 进入 **数据脱敏** → **脱敏任务**
2. 点击 **➕ 创建任务**
3. 选择数据来源（数据集）
4. 为每列手动选择脱敏规则
5. 或点击 **智能推荐** 一键配置

### 5.3 预览确认

**这是最关键的一步！** 系统展示 10-20 条原始数据与脱敏后数据的对比：

| 行号 | 原始值 | 脱敏后 | 规则 |
|------|--------|--------|------|
| 1 | 张三 | 李四 | 姓名仿真 |
| 2 | 13800138000 | 139****8000 | 手机号部分遮盖 |

- ✅ 满意 → 点击"开始脱敏"
- ❌ 不满意 → 返回上一步调整规则

### 5.4 下载和报告

脱敏完成后：

- **📥 下载副本**：下载脱敏后的 CSV/Excel 文件
- **📄 下载报告**：下载 HTML 或 Markdown 格式的脱敏报告
- **👁️ 预览报告**：在浏览器新标签页直接查看 HTML 报告

## 9. AI智能模块

AI模块是本平台的智能化升级功能，基于大语言模型（LLM）实现语义级的敏感信息识别和脱敏，支持 DeepSeek、OpenAI、千问、Kimi、GLM 等多种大模型。

### 9.1 AI配置管理（首次使用必须先配置）

1. 进入 **AI智能** → **AI配置**
2. 点击 **新增配置**
3. 填写配置信息：
   - **配置别名**：如"DeepSeek-V3"
   - **AI提供商**：选择 DeepSeek / OpenAI / 千问 / Kimi / GLM / 文心 / Azure / 自定义
   - **模型名称**：如 deepseek-chat、gpt-4o-mini 等
   - **API密钥**：输入你的 API Key
   - **API地址**：默认即可，自定义模型需填写
   - **联网搜索**：可选开启（需模型支持）
   - **温度**：0.1-1.0，越低越确定
   - **最大Token数**：默认4096
4. 点击 **测试连接** 验证配置是否可用
5. 点击 **保存**，系统会自动激活当前配置

> 💡 配置保存后自动设为激活状态，所有AI任务将使用该配置。

### 9.2 AI敏感识别

AI识别基于LLM的语义理解能力，可以识别出规则引擎难以捕捉的敏感信息。

**操作步骤：**

1. 进入 **AI智能** → **AI识别**
2. 点击 **新建AI识别任务**
3. 填写配置：
   - **任务名称**：可选
   - **选择数据集**：选择已导入的数据集
   - **启用联网搜索**：可选（增强识别准确性）
   - **扫描范围**：全部列/指定列
4. 点击 **开始AI识别**

**识别过程中：**
- 系统逐行逐列调用LLM API进行分析
- 实时显示识别进度
- 点击 **取消任务** 可随时终止

**识别完成后的功能：**

**AI分析结果：**
- 每条识别的敏感信息包含：敏感类型、置信度、风险等级（high/moderate/low）
- 引用法规依据（如PBoC、NFRA等监管机构）
- AI推理过程记录（点击查看详情）

**敏感类型分布统计**：以饼图展示各类敏感数据的占比

**人工复核**：AI识别结果可逐条复核确认
- 确认为敏感（标记✅）或否定（标记❌）
- 填写复核理由
- 支持一键全部采纳/拒绝
- 复核统计：已复核/未复核/准确率

### 9.3 AI脱敏

复核完成后，可基于复核结果执行AI脱敏。

**操作步骤：**

1. 在AI识别任务详情页，点击 **去脱敏**
2. 选择脱敏模式：
   - **遮盖模式**：保留首尾字符，中间替换为*
   - **仿真模式**：生成语义一致但内容不同的合成数据
   - **关联仿真模式**：整行多字段一致性合成
3. 选择输出格式：XLSX / CSV
4. 选择是否使用复核结果（推荐勾选，以复核结果为准）
5. 点击 **开始AI脱敏**

### 9.4 AI报告

脱敏完成后，可查看和下载AI报告：

- **HTML在线预览**：在浏览器中查看完整报告
- **HTML文件下载**：保存到本地
- **Markdown文件下载**：纯文本格式，适合版本管理

报告内容包含：
- 任务基本信息
- 敏感类型分布
- 风险等级分布
- AI识别详情列表
- 脱敏前后对比（预览数据）

## 10. 常见场景操作示例

### 场景一：公司 HR 需要保护员工隐私数据

```
1. 准备 employee.xlsx（含姓名、手机号、身份证、薪资等列）
2. 导入 → 上传 employee.xlsx → 命名"员工信息表"

7. 下载脱敏后的文件分享给外部合作伙伴
```

### 场景二：程序员需要导出测试数据库但不泄漏生产数据

```

2. 创建识别任务 → 自动检测 → 完成
3. 点击"去脱敏" → 为每列选"关联仿真"
4. 选择同一密钥组 → 确保跨表一致性
5. 脱敏后下载副本 → 导入到测试环境
6. 测试环境中关联查询依然正常（张三始终→李四）
```

### 场景三：使用AI大模型智能识别复杂敏感信息

```

3. 进入AI智能 → AI识别 → 新建任务 → 选择数据集
4. 启动AI识别 → 等待LLM逐行分析完成
5. 人工复核：逐条确认AI识别结果
6. 一键跳转AI脱敏 → 选择仿真模式 → 开始脱敏
7. 下载脱敏文件 + 下载AI报告
```

## 11. 常见问题解答

<details>
<summary><b>Q: 支持多大的数据文件？</b></summary>

默认最大 100MB。更大文件建议：
- 使用数据库直接连接导入
- 拆分为多个小文件
- 联系管理员调整配置
</details>

<details>
<summary><b>Q: 识别结果不准确怎么办？</b></summary>

1. 检查语言策略是否正确
2. 在"识别规则管理"中创建自定义正则规则
3. 调整规则集，只保留需要的规则
</details>

<details>
<summary><b>Q: 脱敏后数据还能恢复吗？</b></summary>

- 遮盖模式：**不能恢复**，原始数据已永久替换
- 仿真模式：**不能反向还原**，但相同输入+相同密钥=相同输出
- 建议：脱敏前务必备份原数据，或使用"生成副本"模式
</details>

<details>
<summary><b>Q: 关联仿真是什么意思？</b></summary>

关联仿真确保：同样的原始数据（如"张三"），无论在哪个表中脱敏，都变成同样的假数据（如"李四"）。这样保持了表之间的关联关系不受破坏。需要选择相同的脱敏密钥才能实现。
</details>

<details>
<summary><b>Q: 支持哪些浏览器？</b></summary>

推荐：Google Chrome（最新版）、Microsoft Edge（最新版）、Firefox（最新版）
</details>

<details>
<summary><b>Q: Markdown 报告有什么优势？</b></summary>

- **纯文本**：任何编辑器都能打开
- **版本友好**：适合 Git 版本管理
- **平台兼容**：支持 GitHub/GitLab/Notion 渲染
- **转换灵活**：可转换为 PDF/Word
- **轻量级**：体积小，便于传输存储
</details>
