# 敏感信息脱敏平台 — 产品需求说明书 (PRD)

> **版本：** v1.0  
> **产品名称：** 敏感信息智能识别与脱敏平台  
> **产品类型：** Web 应用（B/S 架构）  

---

## 一、产品概述

### 1.1 产品定位

本平台是一款面向企业数据安全管理的 **智能敏感信息识别与脱敏平台**，提供一站式解决方案：从数据导入、敏感信息自动识别、到智能脱敏处理和报告导出。无需编程基础，普通业务人员即可完成数据安全保护。

### 1.2 目标用户

| 用户角色 | 典型场景 | 核心诉求 |
|---------|--------|---------|
| 数据管理员 | 管理公司数据集、制定脱敏规则 | 批量处理、规则复用 |
| 安全合规人员 | 审计数据安全、生成合规报告 | 审计追溯、报告导出 |
| DBA/开发人员 | 导出测试数据、数据迁移 | 关联仿真、跨表一致性 |
| 管理层 | 查看数据安全大盘 | 可视化、关键指标 |
| 外部合作伙伴 | 获取脱敏后的数据 | 数据可用、隐私不泄露 |

### 1.3 核心价值

- **全流程闭环**：识别 → 脱敏 → 报告，一站式完成
- **多语言智能识别**：支持中/英/日/韩/法/德 6 种语言
- **独创关联造数**：跨表确定性脱敏，保证关联关系不失
- **零代码操作**：全部图形化界面，无需 SQL 或正则经验
- **四套视觉主题**：经典/Vue经典/暗紫/黑金任意切换

### 2.3 数据脱敏

**2.3.1 脱敏策略**
| 策略 | 说明 |
|------|------|
| 完全遮盖 | 全部字符替换为 * |
| 部分遮盖 | 保留首尾、中间替换 |
| 仿真 | 生成同格式但内容不同的假数据 |
| 关联仿真 | 相同原始值 → 相同假数据（跨表一致） |
| 固定遮盖 | 替换为固定字符串 |

**2.3.2 脱敏任务**

## 五、数据库设计

### 5.1 核心表

**datasets** — 数据集
```
id, name, description, source_id, file_path, file_size, 
row_count, column_count, columns(JSON), encoding,
preview_data(JSON), is_active, created_at
```

**detection_tasks** — 识别任务
```
id, name, dataset_id, rule_set_id, scan_columns(JSON),
language_strategy, status, progress, scanned_rows, 
total_rows, found_count, duration_seconds, created_at
```

**detection_results** — 识别结果
```
id, task_id, dataset_id, row_index, column_name,
detected_language, rule_id, rule_name, rule_type,
matched_content, confidence, desensitization_suggestion
```

**desensitization_tasks** — 脱敏任务
```
id, name, dataset_id, detection_task_id, field_rules(JSON),
output_mode, key_id, status, progress, output_path,
duration_seconds, report_path, created_at
```

### 5.2 内置数据

- **脱敏规则 52 条**：遮盖/仿真/关联仿真，覆盖 6 语言
- **脱敏密钥 30 组**：用于业务隔离
- **规则集 2 个**：通用 + 专用场景
