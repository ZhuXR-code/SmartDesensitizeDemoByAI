# 身份证号和国家识别功能增强

## 概述

本次更新大幅增强了敏感数据识别引擎的准确性，主要改进包括：

1. **完整的身份证号校验** - 使用地区码、出生日期、性别、校验位等多维度验证
2. **国家名称精确识别** - 支持98个国家的多语言关键词匹配
3. **地址智能识别优化** - 需要多个关键词或数字+关键词组合才匹配
4. **置信度动态调整** - 根据校验结果自动调整匹配置信度

## 一、身份证号识别增强

### 1.1 校验规则

实现了完整的18位身份证号校验逻辑：

```python
# 校验步骤
1. 格式检查：前17位数字 + 最后1位数字或X
2. 地区码验证：前6位必须是合法的行政区划代码（支持300+地区）
3. 出生日期验证：第7-14位必须是合法日期（YYYYMMDD格式）
4. 年龄合理性：出生年份在1900年至今之间，年龄不超过150岁
5. 性别识别：第17位奇数为男，偶数为女
6. 校验位计算：使用前17位加权求和，取模11得到校验位
```

### 1.2 地区码覆盖

支持全国主要省市自治区的地区码，包括：

- **华东地区**：江苏(32)、浙江(33)、安徽(34)、福建(35)、江西(36)、山东(37)
- **华南地区**：广东(44)、广西(45)、海南(46)
- **华中地区**：河南(41)、湖北(42)、湖南(43)
- **西南地区**：四川(51)、贵州(52)、云南(53)、西藏(54)
- **西北地区**：陕西(61)、甘肃(62)、青海(63)、宁夏(64)、新疆(65)
- **东北地区**：辽宁(21)、吉林(22)、黑龙江(23)
- **华北地区**：河北(13)、山西(14)、内蒙古(15)

每个省份下还包含主要地级市和区县的详细代码。

### 2.2 匹配策略

采用**精确匹配**策略，避免误判：

```python
if rule["id"] == 31:  # 国家名称
    # 按长度降序排序，优先匹配长名称（如"United Kingdom"优于"UK"）
    sorted_keywords = sorted(keywords, key=len, reverse=True)
    for kw in sorted_keywords:
        # 精确匹配整个文本或作为独立词出现
        if text.strip() == kw or re.search(r'\b' + re.escape(kw) + r'\b', text, re.IGNORECASE):
            matched_keyword = kw
            break
```

**匹配条件：**
1. 整个文本完全等于国家名（如："中国"）
2. 国家名作为独立词出现在文本中（使用 `\b` 边界符）

**不匹配的情况：**
- "中国人" - 不是纯国家名
- "美国人" - 不是纯国家名
- "中国制造" - 国家名不是独立词

### 2.3 置信度设置

- 语言设置为 "all"，可匹配任何检测到的语言

## 三、地址识别优化

### 3.1 问题背景

之前的地址识别过于宽松，单个关键词（如"市"、"路"）就会触发匹配，导致大量误判。

### 3.2 新的匹配策略

**中文地址规则（ID=5）：**
- 至少匹配2个地址关键词（如"北京市"+"海淀区"）
- 或者匹配1个关键词 + 包含数字（如"路123号"）

```python
elif rule["id"] in [5, 18, 21, 24, 27, 30]:  # 各种语言的地址规则
    keyword_count = 0
    for kw in keywords:
        if kw.lower() in text.lower():
            keyword_count += 1
    
    # 至少匹配2个关键词，或者包含数字+关键词（如"路123号"）
    has_number = bool(re.search(r'\d+', text))
    if keyword_count >= 2 or (keyword_count >= 1 and has_number):
        matched_keyword = True
```

**支持的地址规则：**
- ID=5: 中文地址关键词（省、市、区、县、街道、路、号、小区、栋、单元、室）
- ID=18: 英文地址关键词（street, avenue, road, boulevard等）
- ID=21: 日文地址关键词（都、道、府、県、市、区、町、村等）
- ID=24: 韩文地址关键词（시, 도, 구, 군, 동等）
- ID=27: 法文地址关键词（rue, avenue, boulevard等）
- ID=30: 德文地址关键词（straße, allee, platz等）

### 3.3 置信度设置

- 地址规则匹配：置信度 0.80（中等置信度）

### 3.4 识别示例

**应该匹配的地址：**

## 四、语言过滤优化

### 4.1 问题

之前国家规则的 `language="all"` 无法通过语言过滤，导致国家名称永远无法被识别。

### 4.2 修复

修改 `scan_text` 方法的过滤逻辑：

```python
# 修改前
if rule["language"] != detected_lang and detected_lang != "unknown":
    continue

# 修改后
# 允许 "all" 语言的规则匹配任何检测到的语言
if rule["language"] != "all" and rule["language"] != detected_lang and detected_lang != "unknown":
    continue
```

这样 `language="all"` 的规则（如国家名称）可以匹配任何语言的内容。

| 0.60 | 很低置信度 | 格式匹配但校验失败的身份证 |

## 八、技术细节

### 8.1 身份证校验算法

**校验位计算公式：**

```python
权重数组: [7, 9, 10, 5, 8, 4, 2, 1, 6, 3, 7, 9, 10, 5, 8, 4, 2]
校验码表: ['1', '0', 'X', '9', '8', '7', '6', '5', '4', '3', '2']

计算步骤:
1. 前17位分别乘以对应权重
2. 求和后取模11
3. 用余数查表得到校验码
4. 与第18位比较
```

**示例：**
```
身份证号: 110101199001011234

计算过程:
1*7 + 1*9 + 0*10 + 1*5 + 0*8 + 1*4 + 1*2 + 9*1 + 9*6 + 0*3 + 0*7 + 1*9 + 0*10 + 1*5 + 1*8 + 2*4 + 3*2
= 7 + 9 + 0 + 5 + 0 + 4 + 2 + 9 + 54 + 0 + 0 + 9 + 0 + 5 + 8 + 8 + 6
= 126

126 % 11 = 5
校验码表[5] = '4'

实际第18位是 '4'，校验通过！
```

### 8.2 国家名称匹配算法

**优先级策略：**
1. 按长度降序排序关键词（优先匹配长名称）
2. 尝试精确匹配整个文本
3. 尝试单词边界匹配（`\b`）

**示例：**
```
输入: "I am from United Kingdom"

匹配过程:
1. 先尝试 "United Kingdom" (16字符) → 匹配成功
2. 不再尝试 "UK" (2字符)

如果使用简单包含匹配，可能会先匹配到 "UK"，导致不准确
```

### 8.3 地址匹配算法

**多条件判断：**
```python
# 条件1: 至少2个关键词
if keyword_count >= 2:
    match = True

# 条件2: 1个关键词 + 数字
elif keyword_count >= 1 and has_number:
    match = True

# 否则不匹配
else:
    match = False
```

**示例分析：**
```
输入: "北京市海淀区中关村大街1号"

关键词匹配:
- "市" ✓
- "区" ✓
- "街" ✓
- "号" ✓

keyword_count = 4 >= 2 → 匹配成功

关键词匹配:
- "市" ✓

keyword_count = 1
has_number = False

不满足任一条件 → 不匹配
```
