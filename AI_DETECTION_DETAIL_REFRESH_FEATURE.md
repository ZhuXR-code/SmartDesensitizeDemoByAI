# AI检测详情页面刷新功能增强

## 📋 功能概述

为AI智能识别的检测详情对话框添加了完善的刷新机制，包括手动刷新、自动定时刷新和任务状态可视化。

## ✨ 新增功能

### 1. **任务状态可视化**
在检测详情对话框顶部添加了醒目的状态标签：
- 🔵 **处理中** - 带加载动画的黄色标签
- 🟢 **已完成** - 绿色成功标签
- 🔴 **已失败** - 红色错误标签
- ⚪ **已取消** - 灰色信息标签

### 2. **失败原因提示**
当任务执行失败时，会显示红色的警告框，清晰展示错误信息：
```
任务失败: [具体的错误原因]
```

### 3. **手动刷新按钮**
- 按钮位置：对话框右上角
- 功能：立即刷新任务状态和检测结果
- 智能禁用：任务完成后自动禁用"刷新状态"模式，改为普通"刷新"
- 加载状态：刷新时显示loading动画

### 4. **自动定时刷新**
- 开关控制：通过切换开关开启/关闭自动刷新
- 刷新频率：每3秒自动刷新一次
- 智能停止：任务完成/失败/取消后自动停止刷新
- 静默更新：自动刷新失败时不弹窗打扰用户（仅控制台输出）
- 资源清理：对话框关闭或组件卸载时自动清理定时器

### 5. **智能行为**
- 打开运行中的任务详情时，自动开启自动刷新
- 任务状态变化时自动刷新任务列表
- 任务结束时显示相应的提示消息

## 🎯 使用场景

### 场景1：监控正在运行的任务
1. 点击"详情"查看正在运行的任务
2. 系统自动开启每3秒刷新一次
3. 实时看到进度、耗时、发现数量等数据更新
4. 任务完成后自动停止刷新并提示

### 场景2：手动检查任务状态
1. 关闭自动刷新开关
2. 需要时点击"刷新"按钮
3. 立即获取最新状态

### 场景3：排查失败原因
1. 任务显示"已失败"状态（红色标签）
2. 下方显示详细的错误信息
3. 根据错误信息进行问题排查

## 🔧 技术实现

### 核心代码结构

```javascript
// 状态管理
const refreshing = ref(false)      // 刷新中状态
const autoRefresh = ref(false)     // 自动刷新开关
const refreshTimer = ref(null)     // 定时器引用

// 关键方法
const refreshDetail()              // 手动刷新
const startAutoRefresh(taskId)     // 开启自动刷新（3秒间隔）
const stopAutoRefresh()            // 停止自动刷新
const toggleAutoRefresh(value)     // 切换自动刷新
const handleDialogClose()          // 对话框关闭清理
```

### 生命周期管理

```javascript
onMounted(() => {
  // 初始化加载
})

onUnmounted(() => {
  stopAutoRefresh()  // 清理定时器，防止内存泄漏
})

// 对话框关闭事件
@close="handleDialogClose"
```

## 📊 用户体验优化

### Before（之前）
- ❌ 无法知道任务是否还在运行
- ❌ 看不到任务失败的原因
- ❌ 需要手动关闭再打开才能刷新
- ❌ 不知道任务什么时候完成

### After（现在）
- ✅ 实时显示任务状态（处理中/已完成/已失败/已取消）
- ✅ 失败时清晰展示错误信息
- ✅ 支持手动刷新和自动刷新两种模式
- ✅ 任务完成时自动提示并停止刷新
- ✅ 资源自动清理，无内存泄漏

## 🔒 资源管理

### 定时器清理策略
1. **对话框关闭时** - `handleDialogClose()`
2. **组件卸载时** - `onUnmounted()`
3. **任务结束时** - 自动检测并停止
4. **切换开关时** - `toggleAutoRefresh()`

确保不会产生内存泄漏和无效的网络请求。

## 📝 注意事项

1. **自动刷新频率**：设置为3秒，平衡了实时性和服务器负载
2. **静默失败**：自动刷新失败时不弹窗，避免打扰用户
3. **智能提示**：只在关键状态变化时显示消息（完成/失败/取消）
4. **按钮禁用**：任务完成后禁用"刷新状态"模式，避免不必要的请求
