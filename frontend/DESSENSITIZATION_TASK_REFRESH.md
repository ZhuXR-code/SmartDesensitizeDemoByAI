# 脱敏任务详情页面 - 实时刷新功能

## 📋 功能概述

为脱敏任务详情页面添加了**手动刷新按钮**和**自动实时刷新**功能，让用户能够实时监控任务进度和状态变化。

## ✨ 新增功能

### 1. 手动刷新按钮

**位置：** 页面右上角，下载按钮左侧

**功能：**
- 点击"刷新"按钮立即重新加载任务信息和结果数据
- 显示加载状态（loading spinner）
- 刷新成功后显示提示消息

**使用场景：**
- 用户想主动查看最新进度
- 任务状态发生变化时手动更新
- 验证操作结果

### 2. 自动实时刷新

**位置：** 页面右上角，手动刷新按钮右侧

**功能：**
- 开启后每3秒自动刷新一次任务状态和进度
- 智能更新策略：
  - **轻量更新**：只更新状态、进度、处理行数等关键指标
  - **完整更新**：当任务从运行状态变为完成状态时，自动加载结果列表
- 关闭后立即停止刷新，避免不必要的API请求

**使用场景：**
- 长时间运行的脱敏任务
- 需要实时监控进度的场景
- 等待任务完成的场景

## 🎯 技术实现

### 核心代码

```javascript
// 手动刷新
const handleManualRefresh = async () => {
  refreshing.value = true
  try {
    await loadTask()  // 完整加载任务和结果
    ElMessage.success('刷新成功')
  } catch (e) {
    ElMessage.error('刷新失败')
  } finally {
    refreshing.value = false
  }
}

// 自动刷新定时器
const startAutoRefresh = () => {
  refreshTimer = setInterval(async () => {
    const res = await getDesensitizationTask(taskId)
    const newTask = res.data || {}
    
    // 只更新关键字段（轻量级）
    task.value = {
      ...task.value,
      status: newTask.status,
      progress: newTask.progress,
      processed_rows: newTask.processed_rows,
      total_rows: newTask.total_rows,
      duration_seconds: newTask.duration_seconds
    }
    
    // 任务完成时加载结果
    if (newTask.status === 'completed' && task.value.status !== 'completed') {
      await loadResults()
      ElMessage.success('任务已完成！')
    }
  }, 3000)  // 3秒间隔
}
```

### 生命周期管理

```javascript
onMounted(() => {
  loadTask()  // 初始加载
})

onUnmounted(() => {
  stopAutoRefresh()  // 组件销毁时清除定时器
})
```

## 📝 注意事项

1. **自动刷新会持续发送API请求**，建议在不需要时关闭
2. **网络较差时**，建议手动刷新而非自动刷新
3. **任务完成后**，自动刷新仍会继续，建议手动关闭
4. **离开页面时**，定时器会自动清理，无需担心内存泄漏
